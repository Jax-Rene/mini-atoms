# mini-atoms Demo 演示脚本（P0）

## 目标

- 演示从自然语言到可运行应用的完整闭环
- 展示发布页与分享页（只读）
- 展示 Timer 原语（前端计时 + 手动保存）

## 演示前准备

1. 打开首页并注册/登录
2. 确认 `DEEPSEEK_API_KEY` 已配置（如未配置会回退 Stub）
3. 准备两个示例需求：
   - Todo：`做一个待办应用，支持新增任务、完成状态切换、数量统计`
   - Focus：`做一个番茄钟专注应用，记录专注时长并统计总分钟数`

## 场景 A：生成 Todo 应用（主闭环）

1. 在 `/projects` 输入 Todo 需求并创建项目
2. 在项目详情页继续输入一条补充需求，点击 `生成草稿`
3. 说明 AI 动画（理解需求 / 生成结构 / 校验 / 准备预览）
4. 生成完成后，展示：
   - 聊天区消息（user + AI）
   - 预览区 `form / list / toggle / stats`
5. 新增一条记录，展示列表和统计同步刷新
6. 点击 `编辑` -> 修改标题/完成状态 -> `保存修改`
7. 点击 `删除`，确认记录消失

## 场景 B：发布与分享

1. 在项目详情页点击 `发布到 /p`
2. 打开 `/p/{slug}` 展示公开应用预览（当前支持写入）
3. 返回项目详情页点击 `生成分享页`
4. 打开 `/share/{slug}` 展示：
   - 聊天记录
   - 草稿/已发布 Spec 面板
   - 只读预览
5. 说明后端只读保护（分享页写接口返回 403）

## 场景 C：Timer 原语（M8）

1. 新建 Focus/番茄钟项目并生成草稿
2. 展示 Timer 面板（工作/休息切换、开始/暂停/重置）
3. 启动计时，暂停
4. 点击 `保存本次专注记录`
5. 展示 `focus_sessions` 列表新增记录、`stats sum(minutes)` 更新

## 演示备注

- DeepSeek 返回格式若有差异，服务端会做 JSON 清洗与字段别名归一化
- 生成失败时不会覆盖旧草稿，并会在聊天区显示错误
- 分享页为严格只读；发布页当前允许写入（roadmap 当前方案）
