# mini-atoms - MVP PRD（收敛版）

版本：v1.1（收敛版）

---

# 1. 产品定位

mini-atoms 是一个 AI 驱动的轻量应用生成平台。

用户通过自然语言描述需求，系统自动：

1. 解析需求
2. 生成结构化应用设计（Spec）
3. 渲染为可运行应用（Spec 驱动，非代码生成）
4. 支持多轮对话迭代（每轮重新生成草稿）
5. 支持发布与分享

产品核心闭环：

> 自然语言 -> 结构化设计 -> 可运行应用 -> 草稿迭代 -> 发布与分享

---

# 2. 目标用户

- 个人开发者
- 产品经理
- 希望快速构建轻量内部工具（CRUD）的用户

---

# 3. MVP 目标（P0）

## 3.1 必须实现

- 简单账号密码注册与登录（本地账号体系，无第三方认证）
- 创建项目 / 打开项目
- 自然语言输入需求
- LLM 生成结构化 Spec（生成草稿）
- 多轮对话支持（每轮对话重新生成完整草稿结果）
- AI 执行过程可视化（简化版，固定阶段展示即可）
- 基于 Spec 渲染可运行应用（非代码生成）
- 通用 CRUD 能力（至少覆盖 List / Form / Toggle / Stats）
- 数据持久化（SQLite）
- 发布功能（维护草稿版本与上线版本）
- Share 分享页面（公开只读，展示对话与当前结果）

## 3.2 范围收敛说明（关键）

- 不做完整版本管理（不做 version 历史列表 / 版本切换 / 回滚）
- 仅维护两种状态：
  - 草稿（Draft）：当前最新生成结果
  - 已发布（Published）：当前上线结果
- 多轮对话每次都重新生成草稿，不做增量 patch 合并

---

# 4. 非目标（P0 不做）

- 第三方认证（OAuth / SSO）
- 完整版本管理（多版本浏览、回滚、差异对比）
- Showcase 内容与页面（仅预留字段与入口位置）
- 多人协作
- 外部 API 集成
- 动态代码生成与执行
- 沙箱进程 / 多进程隔离
- 复杂权限体系

---

# 5. 核心用户流程

## 5.1 创建应用（首次生成）

1. 用户注册/登录
2. 创建项目
3. 输入自然语言需求
4. 系统调用 LLM 生成 Spec
5. 生成草稿 Spec（Draft）
6. 记录本轮对话（用户输入 + AI 回复）
7. 前端展示 AI 忙碌态/逐字动画（不落库）
8. 同步 collections（如有新增）
9. 跳转草稿预览页

## 5.2 多轮迭代（保留）

1. 用户继续输入修改需求
2. 系统拼接历史对话 + 当前草稿 Spec
3. 调用 LLM
4. 重新生成完整草稿 Spec（覆盖当前草稿）
5. 记录本轮对话（用户输入 + AI 回复）
6. 前端展示 AI 忙碌态/逐字动画（不落库）
7. 刷新草稿预览

说明：

- 不创建历史版本列表
- 历史可通过对话记录查看
- 当前产品状态以“草稿 / 已发布”双态为准

## 5.3 发布流程（双态）

1. 用户点击 Publish
2. 将当前草稿 Spec 复制为已发布 Spec
3. 若无发布 slug，则生成 slug
4. 访问路径：

/p/{slug}

说明：

- 每个项目只有一个“当前上线版本”
- 再次发布会覆盖上线内容（slug 保持不变）

## 5.4 分享流程（核心能力，简化版）

1. 用户点击 Share
2. 生成或获取 share_slug
3. 生成访问路径：

/share/{slug}

分享页面包含（P0）：

- 对话记录（按轮次，含用户与 AI 回复）
- 当前草稿预览（只读）
- 当前已发布状态/预览（若已发布）
- Spec JSON（可选展示，便于演示）

无需登录即可查看，且页面为只读。

# 6. AI 工作流可视化（简化版，前端临时效果）

P0 不追求复杂编排与真实执行回放，采用前端临时动画/状态展示：

- 接收需求
- 正在生成（loading / 逐字效果）
- 校验与渲染中
- 完成 / 失败

目的：

- 增强信任感
- 让生成过程可解释
- 便于演示与排错

说明：

- 上述状态用于前端交互反馈，不要求后端持久化存储
- Share 页面不要求回放这些动画状态

---

# 7. 原语能力集（Primitives）

MVP 支持：

1. Navigation
2. List
3. Form
4. Toggle
5. Stats
6. Timer（可先做简化交互版）

说明：

- 新应用 = 原语组合 + Collection 定义
- 不生成代码，只生成可校验的 Spec

---

# 8. 字段类型与数据限制

支持字段类型：

- text
- int
- real
- bool
- date
- datetime（用于计时器/时间点场景）
- enum

限制：

- 单 project collections <= 5
- 单 collection 字段 <= 10
- 不支持嵌套结构
- 不支持数组字段

---

# 9. 草稿 / 发布语义（必须统一）

- 草稿（Draft）：
  - 由 LLM 每轮对话重新生成
  - 用于项目编辑与预览
- 已发布（Published）：
  - 由用户显式点击 Publish 生成
  - 作为公开访问 `/p/{slug}` 的内容
- 数据（records）：
  - P0 按项目维度持久化
  - 草稿与已发布共享同一份项目数据

注意：

- P0 不支持复杂 schema 迁移
- 若草稿引入不兼容 schema 变更，系统可拒绝并提示用户简化/重命名

---

# 10. 安全与边界

- 不执行 LLM 生成代码
- 校验 Spec 合法性（结构 + 字段类型 + 原语白名单）
- 限制字段类型 / collection 数量 / 字段数量
- 限制上下文长度（避免 prompt 失控）
- Share 页面只读（后端接口级保护，不仅是前端禁用）
- 简单账号密码认证（密码哈希存储）

---

# 11. P0 验收标准（可执行）

## 11.1 创建与生成

- 用户可注册/登录
- 可创建项目并输入需求
- 可生成非预置应用（例如 Todo / 读书记录）
- 生成后可看到草稿预览

## 11.2 CRUD 可用

- 可新增记录
- 可查看列表
- 可编辑/切换布尔字段（若 Spec 含 Toggle）
- Stats 可显示基础统计（如 count）

## 11.3 多轮对话可用

- 在同一项目继续输入修改需求
- 系统重新生成草稿并刷新预览
- 历史对话可查看
- AI 忙碌态/逐字动画可查看（前端临时效果）

## 11.4 发布与分享可用

- 点击 Publish 后生成 `/p/{slug}`
- 发布链接可访问
- 点击 Share 后生成 `/share/{slug}`
- Share 页面无需登录可查看
- Share 页面无法触发写操作

## 11.5 演示可交付

- 在线 Demo 可访问
- 至少完成 1 个非预置应用从生成到发布再到分享的完整闭环演示
